# 도시인문학 연구 대시보드 (KCI 논문 분석)

> KCI(한국학술지인용색인)에서 **'도시인문학'** 키워드로 검색된 논문 전체(447편)를 기반으로, 연구 분야의 양적/질적 지형도를 입체적으로 탐색하는 인터랙티브 대시보드입니다.

---

## 📌 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 데이터 범위 | 2003–2025년, 논문 447편, 참고문헌 12,902건 |
| 핵심 학술지 | 도시인문학연구 (243편, 54%) |
| 총 인용 횟수 | 1,491회 |
| 참여 기관 수 | 137곳 |
| 핵심 의의 | 도시인문학이라는 융복합 학문 공동체가 지난 20여 년간 **'어떤 관점(Perspective)'으로 도시를 사유해 왔는지**, 지식 생산의 불균형과 지형도는 어떠한지 시각화 |

---

## ✨ 주요 기능 및 Deep Dive 뷰

차량 계기판 형태의 직관적인 메인 화면과, 개별 지표를 심층적으로 분석할 수 있는 **상세 분석(Deep Dive)** 뷰를 제공합니다.

1. **🔭 인문학의 편중 (Poverty of Humanities)**
   - 양적 다수를 차지하는 '사회·공동체(40%)' 관점과 질적 파급력을 독점하는 '철학·이론(14%)' 관점 간의 불균형을 도넛 차트로 시각화
   - **이론적 토대 네트워크**: 벤야민, 푸코 등 주요 서구 이론가들이 국내 논문에서 '방법론적 도구'로 어떻게 차용(14% vs 86%)되는지 동시 출현 키워드 네트워크(ECharts)로 분석
   - 인용수 5회 이상의 핵심 문헌을 관점 DNA에 따라 자동 배지(초기 담론 형성, 방법론적 렌즈 제공 등) 부여

2. **📍 지역별 연구 방향성과 팩트체크**
   - 8개 광역 권역별 기관 참여도 및 주력 관점 시각화
   - **Fact Check**: "서울 소재 연구자의 지방 연구 비율은 1.3%(6편)에 불과하다"는 인사이트를 도출하여, 학술 블랙홀 서울과 거점 연구 로컬의 양분화 현상 고발 (관련 논문 6편 모달 연동)

3. **🧬 핵심 코어 연구자 패널**
   - 최상위 논문 생산 연구자 10인의 정량적 지표(주저자 비율 등) 제공
   - 파이썬으로 추출한 **소속 기관 및 주요 전공(예: 국어국문학, 신문방송학 등)** 데이터를 이름 옆에 명시하여 도시인문학의 거대한 융복합 생태계 증명

4. **📈 시계열 키워드 진화 & 생태계 (Ecosystem)**
   - 1기(탐색기) → 2기(성장기) → 3기(확장기)로 이어지는 핵심 키워드의 이동을 추적

## 🔍 주요 발견 (Insights)

- **지식 생산의 불균형**: 전체 연구의 40%가 '사회·공동체' 현상에 집중되어 있으나, 정작 학계 내에서 가장 강력한 영향력(인용 최상위권 독식)을 행사하는 것은 단 14%에 불과한 '철학·이론' 관점입니다.
- **서구 이론의 도구화**: 푸코의 '헤테로토피아' 논문(허경, 2011)이 압도적 1위(66회 인용)를 기록했습니다. 하지만 대다수의 이론 논문은 순수 철학적 목적이 아니라, 다른 현상을 설명하기 위한 '방법론적 렌즈'로 차용되고 있음이 수치로 입증되었습니다.
- **서울과 로컬의 시선**: 서울은 전체 논문의 68%를 쏟아내지만 지방 도시에 대한 연구는 거의 하지 않습니다. 반면 비수도권 연구자들은 철저히 자신이 속한 지역을 거점으로 탐구합니다.

## 🛠️ 기술 스택 및 구조

| 분류 | 내용 |
|---|---|
| 프론트엔드 | HTML5, Vanilla CSS, Vanilla JavaScript (ES6+) |
| 차트 & 시각화 | Chart.js 4.4, Apache ECharts 5.5 (네트워크 그래프) |
| 데이터 전처리 | Python 3 (pandas, openpyxl) — `collect_institutions.py`, `collect_perspectives.py` 등 |
| 빌드 및 배포 | Bash (`build.sh`) 기반 JS/CSS Minify 후 정적 파일 배포 |

별도의 백엔드나 DB 없이 **정적 파일**로만 구동되어 유지보수가 매우 용이합니다. 파이썬 스크립트들이 1.7MB의 KCI 원본 엑셀 데이터를 파싱하여 가벼운 JSON/JS 형태의 데이터 파일로 빌드합니다.

```
Dashboard/
├── Dashboard_Urban Humanities/   # 프론트엔드 배포 폴더
│   ├── index.html                # 대시보드 메인 HTML
│   ├── style.css / style.min.css # 전역/컴포넌트 스타일 (Light Theme)

│   └── README.md
├── processed/                    # 파이썬 파싱 중간 단계 CSV 파일들
├── *.py                          # 데이터 추출/정제 파이썬 스크립트
├── build.sh                      # 프론트엔드 Minify 빌드 쉘 스크립트
├── project note.md               # 데이터 분석 방법론 및 의사결정 기록 (매우 중요)
└── .gitignore                    # 민감 파일(API KEY, 원본 데이터 등) 차단
```

> **원본 데이터 (`main data.xlsx`) 및 API 키 폴더는 보안 및 저작권상 저장소에 포함되지 않습니다.**

## 🚀 실행 방법

웹 브라우저에서 `index.html`을 바로 열어 실행할 수 있습니다. (Chart.js 등 외부 라이브러리 로드를 위해 **인터넷 연결** 필수)

```bash
# macOS
open "Dashboard_Urban Humanities/index.html"

# 또는 로컬 서버로 실행 시
cd "Dashboard_Urban Humanities"
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```
